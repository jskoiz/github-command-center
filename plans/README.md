## Direction options

- **Expose scan scope as a dashboard control**: `scanLimit` already exists in
  `server/github-dashboard.ts:126` and `vite.config.ts:19`, but `App.tsx` always
  fetches fixed URLs. This would let the operator choose speed versus coverage.
- **Join Actions billing back to repositories**: billing is already aggregated
  by repo in `server/github-dashboard.ts:425` and typed in `src/types/github.ts:90`,
  but the repo table and billing rail do not surface repo-level cost together.
- **Make workflow failures diagnostic**: the app fetches runs and shows failure
  links, but does not fetch failed job names or reasons. This is useful, but it
  adds API latency and needs careful handling of log/annotation sensitivity.
- **Decide hosted-access end state**: README says local-first but also hints at
  an authenticated hosted route. This should be a design spike, not an
  opportunistic refactor.

## Findings considered and rejected or deferred

- **Move dashboard DTOs out of `src/types`**: valid architecture cleanup, but
  lower leverage until tests exist and a server extraction is actually planned.
- **Split `server/github-dashboard.ts` by responsibility**: real debt, but risky
  before Plan 001. Defer until behavior tests cover payload assembly and cache
  semantics.
- **Co-locate repo table columns and sort logic**: valid, but less urgent than
  cache/security/status fixes.
- **Make repo-name shortening viewer-aware**: valid small correctness cleanup
  (`src/lib/format.ts:87` hardcodes `jskoiz`), but lower impact for the current
  personal app than the selected cache/API/CI findings.
- **Document the scan window only**: correct, but Plan 005 should update docs
  while changing quick/full scan semantics.
- **Untangle `shadcn` CLI CSS dependency**: real dependency hygiene, but it
  touches global styling and should wait for visual regression coverage.
- **Collapse `react-icons` for one GitHub glyph**: valid small dependency
  cleanup, but not worth spending an early executor slot.
- **Stop suppressing TypeScript 6.0 deprecation warnings**: worth checking later;
  not selected because current lint and no-emit typechecks pass.
- **Radix umbrella imports**: possible install/bundle improvement, but production
  bundle impact was not measured in this read-only audit.
