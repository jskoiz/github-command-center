## Direction options

- **Hosted hardening pass**: Plans 006-008 together form the minimum hosted
  hardening sequence before inviting outside users.
- **Scan scope and completeness controls**: Plan 009 makes partial data visible;
  a later direction plan could expose scan scope as a user control.
- **Workflow diagnostics**: Plan 010 reports partial fetch failures. A later
  feature could fetch failed jobs or annotations, but that should be weighed
  against API cost and log sensitivity.

## Findings considered and rejected or deferred

- **Split `server/github-dashboard.ts` by responsibility**: real debt, but
  lower leverage than hosted boundary tests and security hardening.
- **Split `RepoTable.tsx` by responsibility**: valid component debt, but it is
  not blocking the selected correctness/security work.
- **Clean up Node experimental localStorage test warnings**: the test suite
  passes; defer unless the warning starts hiding real failures.
- **Add hosted CI workflows**: out of scope because this repo can be verified
  locally, and hosted Apple-runner budget rules should not be touched here.
- **Build a full distributed session store now**: not justified by the current
  single-process hosted server; Plan 007 names it as a future migration trigger.
